# 设备延保策略智能生成系统 —— 硕士论文完成计划（200h · 50 周）

**论文题目**：设备延保策略的智能生成：数据驱动与大模型推理的融合研究
**研究生**：庄为维（在职）
**总体预算**：200h / 50 周（平均 4h/周）
**核心重点**：**系统数据库表设计**（作为论文工程化主贡献）
**起始日期**：2026-05-11

## 一、总体框架与取舍原则

### 1.1 节奏原则

| 原则 | 说明 |
|---|---|
| 不均匀投入 | 冲刺周 6–8h / 常规周 4h / 休息周 0–2h，允许生活与工作波动 |
| 文献常驻 | 从 W1 起每周 1h 精读 1 篇，累计覆盖 5 个主题约 40 篇 |
| 数据库优先 | 前 16 周把数据库表设计做到答辩级，锁死核心资产 |
| 论文前置化 | 从 W16 就开始写论文章节，不等全做完再动笔 |
| 最小可跑 | Agent/插件只做"能跑通 1 个 case"，不追求工程完备 |

### 1.2 取舍原则（相比 253h 原计划的压缩依据）

| 做深（论文重点） | 做浅（方案+最小 demo） | 硬砍（不做） |
|---|---|---|
| 数据库表设计（ER/DDL/字典/索引） | Agent 端到端 1 个 case | 自研 Agent 框架 |
| 可靠性理论 + Weibull 拟合 | 多目标优化（加权法示例） | NSGA-II 完整实现 |
| RAG 最小闭环 + 对比实验 | 市场情报（mock 数据） | 真实企业数据对接 |
| 合规红线机制 | 博弈回测（方法描述 + 小样例） | 全量法规条文算子化 |
| Agent 评价矩阵 | 分级记忆（落库即可） | 前端 UI / 容器化部署 |

### 1.3 六阶段时间分配

| 阶段 | 时长 | 周数 | 占比 | 核心交付 |
|---|---|---|---|---|
| 阶段 0：前置决策与文献启动 | 6h | W1–W2 | 3% | 选型定稿 + 文献计划 |

| 阶段 4：决策工具插件 | 20h | W33–W36 | 10% | 三大插件 + Pareto 示例 |

## 二、阶段 0：前置决策与文献启动（W1–W2 · 6h）

在动表之前必须先定四件事，否则 A 阶段会反复返工。

| 决策项 | 推荐方案 | 理由 | 备选 |
|---|---|---|---|
| 数据库选型 | **PostgreSQL + pgvector** | 关系 + 向量一库搞定；省去双库同步；论文表述简洁 | MySQL + Milvus |
| Agent 框架 | **LangGraph** | 状态机天然映射 A4 决策域表；节点日志可直接入库 | Dify（零代码快但定制弱） |
| 贯穿案例 | **一台工业注塑机 / 数控机床的 3 年延保询价** | 数据可合成；涉及高价值设备，风险/法律/精算都能展开 | 家电延保（太浅）、新能源车电池延保（数据敏感） |
| 数据来源 | **NASA C-MAPSS + 合成延保合同** | 公开数据集，论文可复现；合成合同可由 LLM 批量生成 | PHM08/12 |

| 任务 | 时长 | 产出 |
|---|---|---|
| W1-①：数据库/框架/案例三项选型 + 与导师对齐 | 2h | 《选型决策备忘》1 页 |
| W1-②：文献清单制定（5 主题 × 8–10 篇） | 2h | 文献 Excel 清单（题目/作者/年份/摘要/优先级） |
| W2-①：Zotero/Notion 文献管理搭建 + 启动首批精读（2 篇） | 2h | 文献笔记模板 v1 |

**文献 5 主题**：① 延保服务与售后运维；② 可靠性工程与精算；③ LLM Agent 架构（ReAct/CoT/Reflexion）；④ RAG 与知识库动态更新；⑤ 多目标优化与博弈决策。
