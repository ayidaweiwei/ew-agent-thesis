# Markdown 格式优化 Skill

## 触发条件

用户要求优化、格式化、美化、规范化 Markdown 文件时触发。

## 优化规则

### 1. 标题规范

- 标题前后各空一行（文件首行标题除外）
- 标题层级递进，不跳级（如 `##` 后面不能直接跟 `####`）
- `#` 与标题文字之间有且仅有一个空格
- 同级标题风格统一

### 2. 列表规范

- 无序列表统一使用 `-` 符号
- 列表项之间不加空行（除非列表项包含多段落）
- 嵌套列表缩进 3 个空格
- 有序列表从 1 开始递增
- 列表前后各空一行

### 3. 空行规范

- 段落之间有且仅有一个空行
- 不出现连续两个以上空行
- 文件末尾保留一个换行符
- 代码块、表格、引用块前后各空一行

### 4. 中英文排版

- 中文与英文之间加一个空格
- 中文与数字之间加一个空格
- 英文标点后加空格，中文标点后不加空格
- 专有名词大小写正确（如 GitHub、Python、MySQL）

### 5. 表格规范

- 表头与分隔行对齐
- 单元格内容左对齐（除非有特殊需要）
- 表格前后各空一行

### 6. 代码块规范

- 行内代码用单反引号 `code`
- 多行代码用三反引号并标注语言
- 代码块前后各空一行

### 7. 链接与图片

- 链接文字有意义，避免"点击这里"
- 图片添加 alt 文字描述

### 8. 强调规范

- 加粗用 `**文字**`
- 斜体用 `*文字*`
- 不混用 `__` 和 `**`

## 执行步骤

1. 读取目标 Markdown 文件
2. 按上述规则逐项检查并修复
3. 保持原文内容语义不变，只调整格式
4. 输出修改摘要（修复了哪些问题）

## 注意事项

- 不修改文件的实际内容和语义
- 保留用户有意为之的特殊格式（如诗歌排版）
- 如果文件包含 YAML frontmatter，保持其格式不变
- 对于学术论文类 MD，保留其章节编号格式
- 技术路线使用表格
- 分一个step，或者子步骤需要换行和使用不一样的格式
